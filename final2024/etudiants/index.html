Mobile Avancé Examen Final A24

Documentation autorisée: Tout ce que vous avez produit pendant le cours, ainsi que les repository utilisés et présentés pendant le cours.
Non autorisée: Clé USB, Internet (à l'exception du site du cours et de la documentation Flutter), Intelligences Artificielles.

Durée: Vous avez 2 heures pour compléter l'examen. Plus si vous avez le droit à du temps supplémentaire.

Firestore

Faire une copie du projet firebase_start_final et renomme le dossier exo_firestore_nomdefamille. Le projet est connecté à un projet firebase.

On veut deux boutons au milieu de l'application nommés "Obtenir" et "Mettre à jour".

Il y a deux éléments de texte sous le bouton qui permettent d'afficher les informations reccueillies.

Le datastore Firebase contient une collection "etudiants", dont les ID sont les matricules de tout les étudiants du cours.

Chaque entrée contient 3 champs : "nom", "prenom" et "complete"

2 points À l'appui sur le bouton "Obtenir", vous devez obtenir le document lié à votre matricule Ex: 1525462 Vous devez afficher les champs "nom" et "prenom" dans les 2 champs texte sous les boutons.

1 point À l'appui sur le bouton "Mettre à jour", le document ayant pour Id votre matricule doit être mise à jour pour que la propriété "complete" passe de false à true.

HTTP JSON et erreurs

Dans un dossier séparé exo_erreurs, vous devez programmer une interface avec 2 champs de saisie et un bouton.

Le serveur déployé : 

https://examen-final-a24.azurewebsites.net/Exam2024/Division/{dividende}/{diviseur}
Exemple d'usage : 
https://examen-final-a24.azurewebsites.net/Exam2024/Division/100/5
Le webservice vous permet d'effectuer des divisions.
  • Si le diviseur est 0, vous recevrez un message d'erreur (code 400).
  • Si la division réussie, vous recevez un objet json qui va comme suit {"resultat" : ResultatCommeNombreEntier}. Ex : {"resultat" : 20} (code 200).

  • 1 point L'appui sur le bouton envoi une requête GET à l'URL du serveur déployé. Remplacez {dividende} et {diviseur} par le contenu des champs texte.
  • 1 point Si tout se passe bien (aucune erreur) affichez un SnackBar indiquant le message renvoyé (le contenu du champ resultat).
  • 1 point Si l'application n'arrive pas à communiquer avec le serveur, affichez un SnackBar avec un message clair et en français.
  • 1 point Vous affichez le message d'erreur retourné par le serveur dans un Snackbar.
